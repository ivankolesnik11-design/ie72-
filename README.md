
# 🌡️ Thermal Vision Simulator (Симулятор Тепловизора)

Приложение на Python с графическим интерфейсом (Tkinter), симулирующее работу тепловизионной камеры для мониторинга состояния здоровья людей и сельскохозяйственных животных.

Проект демонстрирует логику обработки данных с датчиков температуры, визуализацию через цветовые градиенты и систему рекомендаций based on медицинских норм.

## ✨ Возможности

*   **Два режима работы:**
    *   👤 **Человек:** Мониторинг температуры тела с рекомендациями (врач/измерение).
    *   🐄 **Животное:** Поддержка коров, кур, баранов и свиней с индивидуальными порогами норм.
*   **Визуализация данных:**
    *   Объекты отмечаются зелеными маркерами.
    *   **Цветовой градиент:**
        *   🔵 **Синий:** Температура ниже нормы.
        *   🟢 **Зеленый:** Норма.
        *   🔴 **Красный:** Температура выше нормы.
    *   Отображение точного значения температуры рядом с объектом.
*   **Умные рекомендации:**
    *   Динамический вывод статуса сверху экрана ("С вами все хорошо", "Обратитесь к врачу" и т.д.).
    *   Обработка отсутствия датчика (вывод `Error` и `...`).
*   **Симуляция:**
    *   Генерация случайных объектов в кадре.
    *   Эмуляция показаний датчика температуры.

## 📋 Требования

Для работы приложения необходим Python 3.x и библиотека Pillow.

1.  Установите зависимости:
    ```bash
    pip install pillow
    ```
    *(Библиотека `tkinter` обычно встроена в стандартную установку Python)*

## 🚀 Запуск

1.  Клонируйте репозиторий или скачайте файл `thermal_app.py`.
2.  Запустите приложение из терминала:
    ```bash
    python thermal_app.py
    ```

## 🎮 Как пользоваться

1.  **Главное меню:** Выберите объект наблюдения — **Человек** или **Животное**.
2.  **Если выбрано животное:** Выберите конкретный вид (Корова, Курица, Баран, Свинья). Для каждого вида установлены свои медицинские нормы температуры.
3.  **Экран камеры:**
    *   На черном фоне появятся точки, имитирующие обнаруженные объекты.
    *   Цвет обводки точки покажет состояние: синий (холодно), зеленый (норма), красный (жарко).
    *   Сверху экрана появится текстовая рекомендация.
4.  **Навигация:** Используйте кнопку "Вернуться в меню" или "Назад" для смены режима.

## ⚙️ Технические детали

### Пороги температур (°C)
В коде используются следующие константы (на основе открытых медицинских данных):

| Объект | Мин. норма | Макс. норма |
| :--- | :---: | :---: |
| **Человек** | 35.0 | 37.5 |
| **Корова** | 37.5 | 39.5 |
| **Курица** | 40.0 | 42.0 |
| **Баран** | 37.5 | 39.5 |
| **Свинья** | 38.0 | 40.0 |

### Симуляция датчика
Переменная `self.is_simulating_sensor` в классе `ThermalApp` управляет режимом работы:
*   `True` (по умолчанию): Генерируются случайные температуры вокруг нормы.
*   `False`: Вместо температуры выводится `Error`, а статус заменяется на многоточие `...`.

> **Примечание:** Это приложение является симулятором. Обычные веб-камеры не имеют инфракрасных датчиков температуры. Для реальной работы требуется подключение специализированного тепловизионного оборудования (например, FLIR) и интеграция с их SDK.
